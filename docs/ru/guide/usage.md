# Использование

Подробный разбор каждой страницы, вкладки и элемента управления SKOPOS. Скриншоты — реальный интерфейс; ваши метрики флота будут другими.

## Ежедневный цикл

1. **Аналитика** — трафик, фильтры, период (время в **UTC**).
2. **AI-брифинг экосистемы** — здоровье флота простым языком.
3. **Скор безопасности** и баннер алертов в сайдбаре.
4. **Безопасность** — скор, порты, стуки, аудит, **Сводный отчёт**.
5. **История сканов** — сравнение снимков и регрессии.
6. **AI-агент** (чат справа внизу) — вопросы по алертам и hardening.

## Общая оболочка (на каждой странице)

После входа на любой странице доступны:

- **Сайдбар** — 6 страниц, язык и тема, кольцо security score, баннер алертов, подписи авто-скана/Telegram, выход.
- **Верхнее меню** — **Документация**, **Быстрый старт**, **Настройки** (выравнивание слева).
- **Баннер угроз** — до пяти critical/high алертов со ссылкой на **Безопасность**.
- **Плавающий агент** — кнопка чата справа внизу на любой странице.

## Мастер «Быстрый старт»

Сайдбар → **Быстрый старт** или верхнее меню. Шесть шагов для первого запуска:

1. **Старт** — чеклист: сервер, SSH, пароль, AI-ключ, сбор трафика, security-scan.
2. **Сервер** — первый хост (имя, host, user, port, путь к nginx log) → пишет `servers.yaml`.
3. **SSH** — генерация Ed25519, тест соединения, загрузка public key на хост.
4. **Безопасность и AI** — пароль дашборда, время сессии, OpenRouter, авто-скан.
5. **Сбор** — первый collect логов из мастера.
6. **Скан** — первый security-scan; **Готово** — ссылки на Аналитику, Безопасность, Настройки.

> Мастер можно пропустить — те же опции в **Настройках**. Незавершённая настройка показывает баннер в сайдбаре.

## Страница «Аналитика»

Главная (`dashboard.py`). Сбор access-логов nginx по SSH и дашборды трафика.

### Панель периода и сайдбар

- **Период** — пресеты (день / неделя / месяц / 3 мес. / год) или свой диапазон. Все метки времени — **UTC**.
- **Фильтры** — скрыть ботов, служебный трафик, только внешние IP; multiselect server/host/country; подстрока path.
- **Сайдбар** — путь к `servers.yaml`; **Собрать сейчас** (инкремент) и **Backfill all** (полная история).

### AI-брифинг экосистемы

- Карточка вверху — mood флота, security score, статус коллекторов, главные риски.
- Использует `OPENROUTER_API_KEY`; без ключа — текст по правилам.
- Кэш ~15 минут; кнопка обновления — перегенерация.

![Карточка AI-брифинга экосистемы](../../screenshots/ai-briefing-card.png)


### Вкладки аналитики

### Обзор
- Таймлайн трафика (час или день).
- Donut по странам — запросы vs уникальные посетители.
- Bar charts топ страниц и хостов.
- Heatmap по часам (день × час).
- Распределение HTTP status codes.
### География
- Карта мира — метрика (requests / visitors); опционально **3D globe** и fullscreen.
- Bar/donut/timeline по странам.
- Cross-chart страны × хосты и сводная таблица.
- Нужна GeoIP база (`geoip_mmdb_path`).
### Аудитория
- Donut: браузеры/clients, ОС, устройства.
- Bar chart топ IP посетителей.
### Контент
- Treemap страниц (host → path).
- Популярные paths и hosts.
- Сегменты экосистемы по трафику.
### Источники
- Топ referer domains.
- Метрики direct vs referred.
### Журнал
- Журнал визитов — time, host, server IP, visitor IP, country, client, OS, device, method, path, status, referer (до 1000 строк).
### Система
- Статус коллектора по серверам — last OK, rows inserted, last error.
- Источники логов (nginx files, Apache, docker container IDs).

## Центр безопасности

Сайдбар → **Безопасность**. SSH-пробы: порты, firewall, ресурсы, auth logs, Docker.

- Переопределение пути `agent.yaml`.
- Фильтр по серверу или **Сканировать все серверы** (кнопка в сайдбаре).

KPI: **Находки**, **Критично**, **Высокий**, **Публичные порты** — суммарно по выбранному фильтру.

### Вкладки безопасности

### Скор и алерты
- Fleet score /100, буквенная оценка, progress bar.
- Карточки score по каждому серверу.
- Audit remarks по правилам.
- Список алертов — severity, сообщение, рекомендация.
### Сводный отчёт
- Бейдж риска (Критический / Высокий / Средний / Низкий) и fleet score.
- **LLM-провайдер** — из `agent.yaml` (по умолчанию OpenRouter).
- **Сгенерировать ИИ-отчёт** — полный markdown-анализ; до этого — **предпросмотр на правилах**.
- Метрики: всего находок, critical, high, серверов в отчёте.
- **Скачать Markdown** — для тикетов и runbook.
- Разделы: краткое резюме, периметр, hardening проекта, заметки по серверам.
![Сводный отчёт — AI security brief](../../screenshots/security-summary-report.png)
### Обзор
- Fleet chart по всем серверам.
- Expander на сервер: CPU/memory/load, findings bar, uptime, kernel snippet.
### Порты
- Визуальная карта портов.
- Таблица: proto, port, address, bind scope, exposure (Open/Localhost/Other), process.
- Raw `firewall_status`.
### Стуки в порты
- KPI: события, уник. IP, главная цель, high-threat count.
- Charts: top actors, типы, по портам, страны, timeline, heatmap.
- Таблица акторов — IP, country, classification, threat score, hits, ports, servers.
- Журнал событий (300 последних) — SSH probes, firewall, fail2ban, web scans.
### Ресурсы
- На сервер: gauges CPU %, memory %, load/cores %; network I/O; disk usage.
### Аудит
- Findings по правилам — severity, detail, recommendation.
- Sample auth log при failed SSH logins.
### 3D-карта угроз
- Fullscreen 3D threat map на сервер.
- Легенда: server (center), internet (top), public ports, localhost, findings на внешнем кольце.
![3D-карта угроз](../../screenshots/security-3d-map.png)
### AI-агент (вкладка)
- One-shot **Run AI audit** (отдельно от плавающего чата).
- Выбор провайдера и markdown-результат.
- Для диалога — плавающий агент или **Сводный отчёт**.


## История сканов

Сайдбар → **История сканов**. Для Compare нужно минимум два скана.

- Слайдер окна истории — 7–90 дней.
- Фильтр сервера — все или один хост.

### Вкладки

- **Таймлайн** — Score во времени, календарь активности, fleet radar.
- **Тренды** — Findings по severity во времени.
- **Сравнение** — Scan A vs B — diff chart, новые и закрытые issues.
- **Журнал** — Таблица — time, server, findings, critical, high.

## Страница «Настройки»

Сайдбар или верхнее меню → **Настройки**. Вся конфигурация флота и дашборда.

- **Доступ к дашборду** — Установить/сменить/сбросить пароль, время сессии, чеклист политики.
- **База данных** — SQLite vs PostgreSQL; тест; apply URL и миграция.
- **Авто-скан** — Toggle; интервал 5–1440 мин (default 60).
- **Telegram** — Enable, chat ID, bot token env, интервал, test send после сканов.
- **SSH-ключи** — Статус ключа; генерация Ed25519/RSA.
- **Серверы флота** — Expander на сервер — SSH, nginx/Apache/docker logs, test SSH, upload key, custom commands (`SKOPOS_ALLOW_CUSTOM_SSH_COMMANDS=1`).
- **Добавить / Сохранить** — Форма нового хоста; **Сохранить servers.yaml** пишет config + `.env` tokens.

## Плавающий AI-агент

Кнопка чата справа внизу на каждой странице. DevOps & security ассистент с контекстом флота.

- Выбор провайдера (из `agent.yaml`).
- **Быстрые вопросы** — critical алерты, открытые порты, SSH hardening, port knock, топ рекомендации.
- История чата в сессии; **Очистить** — сброс thread.
- Контекст: posture, scans, traffic summary (`agent/context.py`).

![Плавающий агент безопасности](../../screenshots/floating-agent.png)


## Три AI-интерфейса — когда что использовать

| Surface | Use when |
|------|--------|
| **AI-брифинг** (Аналитика) | Ежедневный абзац о здоровье флота; авто-обновление. |
| **Сводный отчёт** (Безопасность → вкладка) | Формальный markdown remediation brief; скачиваемый. |
| **Плавающий агент** (везде) | Интерактивный Q&A по алертам, портам, hardening. |
| **Безопасность → AI Agent** | One-shot полный audit; не чат. |

## CLI

```bash
python skoposctl.py collect
python skoposctl.py collect --full
python skoposctl.py security-scan
python skoposctl.py discover
```

Ссылки в шапке — **Документация**, **Быстрый старт**, **Настройки** — быстрая навигация без прокрутки сайдбара.
